# 憲法

- すべてのタスクは本憲法を最初に参照してから実行されるものとする。
- 本憲法を参照せずに行われるタスクは以後、存在しない。
- 「rule.md」は初回のみ作成されるファイルである。
- 「rule.md」はユーザーの指示なしに決して改変、削除を行わないこと。
- 「rule.md」を改変、削除するユーザーの指示とは「憲法を改正したいです」という文字列を指し、それ以外に存在しない。
- rodemap.mdに従うこと。
- Todoリストの遂行の直前にはユーザーの許可を得ること。
- 「ユーザーの許可」と判断してよい文字列は「はい」という一単語のみであり、それ以外に存在しない。
- 「ユーザーの否認」と判断してよい文字列は「いいえ」という一単語のみであり。それ以外に存在しない。
- ユーザーに許可を求めた際、「はい」or「いいえ」以外の曖昧な回答や、回答の中に余分な文字列が混在していた場合、todoリストの遂行は不可能と判断し、ユーザーに再び「はい」or「いいえ」の選択を迫ること。
- todoリスト遂行に関してユーザーに許可を求める時、その目的や背景について説明責任を果たすこと。
- 原則、推論は行わない。
- 推論が許可されるケースは、ユーザーが「推論を許可する」という文字列を送信した場合のみであり、それ以外に存在しない。
- 原則、コードの生成や修正、リファクタリングを行うにあたって、使用技術の公式リファレンス、公式マニュアル、公式ドキュメントを一次ソースとして扱うこと。
- 下記のエンジニアコミュニティプラットフォームの情報は2次ソースとして扱うこと。
  - Qiita
  - Zenn
  - StackOverflow
  - GitHubIssue
  - Medium
  - サムライエンジニア
  - その他非公式のブログ記事
- 環境変数ファイル、その他の重要なシステムファイルはセキュリティの観点からGitHubのリモートリポジトリに決してpushしない。
